Scenario is used to describe a tax scenario of a document based on the combination of document type and tag used.

## Schema ID

`https://gobl.org/draft-0/tax/scenario`

## Properties

| Title | Property | Type | Description |
|-------|----------|------|-------------|
| Name | `name` | [i18n.i18n.String](/draft-0/i18n/i18n._string) | Name of the scenario for further information. |
| Description | `desc` | [i18n.i18n.String](/draft-0/i18n/i18n._string) | Description of the scenario for documentation purposes. |
| Type | `type` | array of [cbc.cbc.Key](/draft-0/cbc/cbc._key) | Type of document, if present. |
| Tags | `tags` | array of [cbc.cbc.Key](/draft-0/cbc/cbc._key) | Array of tags that have been applied to the document. |
| Tax Categories | `cat` | array of [cbc.cbc.Code](/draft-0/cbc/cbc._code) | Categories is an optional list of tax category codes that acts as a filter. When set, at least one of the specified categories must be present in the document's line taxes for the scenario to match. |
| Extension Key | `ext_key` | [cbc.cbc.Key](/draft-0/cbc/cbc._key) | Extension key that must be present in the document. |
| Extension Code | `ext_code` | [cbc.cbc.Code](/draft-0/cbc/cbc._code) | Extension code that along side the key must be present for a match to happen. This cannot be used without an `cbc.Code`. The value will be copied to the note code if needed. |
| Note | `note` | [tax.tax.Note](/draft-0/tax/tax._note) | A note to be added to the document if the scenario is applied. |
| Codes | `codes` | [cbc.cbc.CodeMap](/draft-0/cbc/cbc._code_map) | Codes is used to define additional codes for regime specific situations. |
| Extensions | `ext` | [tax.tax.Extensions](/draft-0/tax/tax._extensions) | Ext represents a set of tax extensions that should be applied to the document in the appropriate "tax" context. |
