CorrectionOptions defines a structure used to pass configuration options to correct a previous invoice.

## Schema ID

`https://gobl.org/draft-0/bill/correction-options`

## Properties

| Title             | Property            | Type                                     | Req. | Description                                                                                           |
|-------------------|---------------------|------------------------------------------|------|-------------------------------------------------------------------------------------------------------|
| Issue Date        | `issue_date`        | [cal.Date](/draft-0/cal/date)            |      | When the new corrective invoice's issue date should be set to.                                        |
| Stamps            | `stamps`            | array of [cbc.Stamp](/draft-0/cbc/stamp) |      | Stamps of the previous document to include in the preceding data.                                     |
| Credit            | `credit`            | boolean                                  |      | Credit when true indicates that the corrective document should cancel the previous document.          |
| Debit             | `debit`             | boolean                                  |      | Debit when true indicates that the corrective document should add new items to the previous document. |
| Reason            | `reason`            | string                                   |      | Human readable reason for the corrective operation.                                                   |
| Correction Method | `correction_method` | [cbc.Key](/draft-0/cbc/key)              |      | Correction method as defined by the tax regime.                                                       |
| Corrections       | `corrections`       | array of [cbc.Key](/draft-0/cbc/key)     |      | Correction keys that describe the specific changes according to the tax regime.                       |
